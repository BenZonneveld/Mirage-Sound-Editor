00076 // CPianoCtrl class
00077 //
00078 // This class represents a piano keyboard. 
00079 //---------------------------------------------------------------------
00080 //
00081 // To use the CPianoCtrl class in a dialog based application, use the 
00082 // following steps:
00083 //
00084 //  1. In the resource editor, place a custom control onto the dialog
00085 //     box.
00086 //
00087 //  2. Set the class name in the custom control's property box to match 
00088 //     the CPianoCtrl's class name.
00089 //
00090 //  3. Add a CPianoCtrl instance variable to the dialog class.
00091 //
00092 //  4. Add a DDX_Control call in the DoDataExchange method in the dialog 
00093 //     class. For example, if your dialog class was named CPianoDlg, the 
00094 //     CPianoCtrl variable was named m_PianoCtrl, and the Id for the
00095 //     control was IDC_PIANOCTRL, you would place the following in the
00096 //     dialog class' DoDataExchange function:
00097 //
00098 //      void CPianoDlg::DoDataExchange(CDataExchange* pDX)
00099 //      {
00100 //          CDialog::DoDataExchange(pDX);
00101 //          //{{AFX_DATA_MAP(CPianoDlg)
00102 //          // NOTE: the ClassWizard will add DDX and DDV calls here
00103 //          //}}AFX_DATA_MAP
00104 //          DDX_Control(pDX, IDC_PIANOCTRL, m_PianoCtrl);
00105 //      }
00106 //
00107 //  5. In the dialog class' OnInitDialog function, initialize the 
00108 //     CPianoCtrl object by calling its Initialize function. Here, you 
00109 //     will pass the desired note range and optionally the note-on 
00110 //     color. 
00111 //
00112 // To use the control dynamically within a dialog box or within a window,
00113 // use the following steps:
00114 //
00115 //  1. Add a CPianoCtrl instance variable to your class. This can be a 
00116 //     pointer to a CPianoCtrl, but if so, you will need to allocate
00117 //     memory for it before using it and deallocate its memory after you
00118 //     are done with it.
00119 //
00120 //  2. Call the CPianoCtrl object's Create function. Here, you will 
00121 //     pass the parent window, the CPianoCtrl's position and size, its
00122 //     Id, and optionally its window style.
00123 //
00124 //  3. Call the CPianoCtrl object's Initialize function. Here, you will 
00125 //     pass the desired note range and optionally the note-on color. 
00126 //
00127 //---------------------------------------------------------------------
00128 //
00129 // Some important notes concerning the CPianoCtrl class:
00130 //
00131 //  It can have up to 128 keys (0 - 127). 
00132 //
00133 //  The range is set with the Initialize method or the SetNoteRange
00134 //  method. You specify the lowest note of the range and the highest 
00135 //  note of the range. The lowest note must be less than the highest 
00136 //  note and both the lowest note and the highest note must be natural.
00137 //
00138 //  The number 0 is considered a C note. As you go up from 0, you are 
00139 //  ascending the chromatic scale. Therefore, 0 equals C, 1 equals C#,
00140 //  2 equals D, etc. After you've reached the B note (the first B note 
00141 //  is number 11), the scale starts over from C.
00142 //
00143 //  To be notified of note on and note off events, derive a class from
00144 //  CPianoCtrlListener. Implement its methods in your derived class 
00145 //  and attach an object of that class to a CPianoCtrl object with the
00146 //  AttachListener method. When the CPianoCtrl object receives note on
00147 //  and note off events, it will notify all of its listeners.
00148 //

00327     //------------------------------------------------------------------
00328     // Piano key classes
00329     //
00330     // The following classes represent the keys on the CPianoCtrl class. 
00331     //
00332     //------------------------------------------------------------------
00333     //
00334     // To understand how these classes work, first imagine a piano 
00335     // keyboard display with a range of 12 keys from C to B:
00336     //
00337     //                  ----------------------
00338     //                  | | || | | | || || | |
00339     //                  | |_||_| | |_||_||_| |
00340     //                  |  |  |  |  |  |  |  |
00341     //                  ----------------------
00342     //
00343     // We can divide the keys into 4 types:
00344     //
00345     //                  ----------------------
00346     //                  | | || | | | || || | |
00347     //                  | |_||_| | |_||_||_| |
00348     //                  |  |  |  |  |  |  |  |
00349     //                  ----------------------
00350     //                  /|\      /|\
00351     //                   |        | 
00352     //
00353     //                  White key left (C and F)
00354     //
00355     //
00356     //                  ----------------------
00357     //                  | | || | | | || || | |
00358     //                  | |_||_| | |_||_||_| |
00359     //                  |  |  |  |  |  |  |  |
00360     //                  ----------------------
00361     //                     /|\      /|\ /|\
00362     //                      |        |   |
00363     //
00364     //                White key middle (D, G, and A)
00365     //
00366     //
00367     //                  ----------------------
00368     //                  | | || | | | || || | |
00369     //                  | |_||_| | |_||_||_| |
00370     //                  |  |  |  |  |  |  |  |
00371     //                  ----------------------
00372     //                        /|\          /|\ 
00373     //                         |            |
00374     //
00375     //                  White key right (E and B)
00376     //
00377     //
00378     //                     |  |     |  |  |
00379     //                    \|/\|/   \|/\|/\|/
00380     //                  ----------------------
00381     //                  | | || | | | || || | |
00382     //                  | |_||_| | |_||_||_| |
00383     //                  |  |  |  |  |  |  |  |
00384     //                  ----------------------
00385     //                 
00386     //            Black key (C#, D#, F#, G#, and A#)
00387     //
00388     //
00389     // There is also a fifth type of key. This is a special key used 
00390     // when certain ranges are set. For example, look at a range of keys
00391     // from C to C one octave higher:
00392     //
00393     //                  ------------------------
00394     //                  | | || | | | || || | | | |
00395     //                  | |_||_| | |_||_||_| | |_|
00396     //                  |  |  |  |  |  |  |  |  |
00397     //                  -------------------------
00398     //
00399     // A problem occurs because we have half of a black key dangling off
00400     // the side of the keyboard. This really isn't acceptable, so we 
00401     // need an additional key to take care of this special case, the
00402     // white full key:
00403     //
00404     //                  -------------------------
00405     //                  | | || | | | || || | |  |
00406     //                  | |_||_| | |_||_||_| |  |
00407     //                  |  |  |  |  |  |  |  |  |
00408     //                  -------------------------
00409     //                                        /|\
00410     //                                         |
00411     //
00412     //                        White full key
00413     //
00414     // The white full key takes care of those cases in which there would
00415     // otherwise be a black key chopped in half.
00416     //
00417     // The five types of piano keys are each represented by their own 
00418     // class. Each class knows how to draw itself so that it has the 
00419     // proper shape on the piano keyboard. All of the classes derive 
00420     // from one parent class called CPianoKey. This class provides the 
00421     // interface for all piano key classes.
00422     //
00423     // The CPianoCtrl class only allows ranges that begin and end with 
00424     // natural notes (white keys). This was a judgement call on my part.
00425     // Ranges in which the client chose a sharp/flat key as the lowest
00426     // or highest note would create a strange situation in which a white
00427     // key would be chopped in half:
00428     //
00429     //                    --------------------
00430     //                    | || | | | || || | |
00431     //                    |_||_| | |_||_||_| |
00432     //                    ||  |  |  |  |  |  |
00433     //                    --------------------
00434     //                   /|\
00435     //                    |
00436     //      
00437     //              Half-white key?
00438     //
00439     // This situation could certainly be dealt with, but if the client
00440     // specified that C# should be the lowest key, they would have to 
00441     // deal with the extra half-white key that would actually be the 
00442     // lowest note, in this case C, and not the one they had specified.
00443     // I decided this situation was more trouble than it was worth, so
00444     // I limited the low and high end of the range to natural notes.
00445     //
00446     //------------------------------------------------------------------
00447     //
00448     // The keys are divided into "units." Units are used to determine 
00449     // how large to draw each key. The length of a unit is represented 
00450     // by the total length of the control divided by the number of 
00451     // natural keys in the range divided by 3. 
00452     //
00453     // So if the length of the control is 100 and the number of natural
00454     // keys in the range is 12, than the unit length would be 2.77778.
00455     //
00456     // Here is how the unit length is used:
00457     //
00458     // All of the white keys at their longest part take up 3 units:
00459     //
00460     //                          ----------
00461     //                          | | || | |
00462     //                          | |_||_| |
00463     //                          |  |  |  |
00464     //                          ----------
00465     //                          |<>|<>|<>|
00466     //                          
00467     //
00468     //                         3 Units Long
00469     //
00470     // The black keys are 2 units long and the white keys at their 
00471     // narrowest part are also 2 units long.
00472     //
00473     //                         2 Units Long
00474     //
00475     //                          \/ \/ \/
00476     //                          ----------
00477     //                          | | || | |
00478     //                          | |_||_| |
00479     //                          |  |  |  |
00480     //                          ----------
00481     //
00482     // The unit length is passed to each key object so that it can know
00483     // how large to draw itself. The position on the control is also 
00484     // passed to itself so that it can know where to draw itself.
00485     //
00486     //-----------------------------------------------------------------
