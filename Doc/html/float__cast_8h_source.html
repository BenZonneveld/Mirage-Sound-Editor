00022 /*============================================================================
00023 **  On Intel Pentium processors (especially PIII and probably P4), converting
00024 **  from float to int is very slow. To meet the C specs, the code produced by
00025 **  most C compilers targeting Pentium needs to change the FPU rounding mode
00026 **  before the float to int conversion is performed.
00027 **
00028 **  Changing the FPU rounding mode causes the FPU pipeline to be flushed. It
00029 **  is this flushing of the pipeline which is so slow.
00030 **
00031 **  Fortunately the ISO C99 specifications define the functions lrint, lrintf,
00032 **  llrint and llrintf which fix this problem as a side effect.
00033 **
00034 **  On Unix-like systems, the configure process should have detected the
00035 **  presence of these functions. If they weren't found we have to replace them
00036 **  here with a standard C cast.
00037 */
00038 
00039 /*
00040 **  The C99 prototypes for lrint and lrintf are as follows:
00041 **
00042 **      long int lrintf (float x) ;
00043 **      long int lrint  (double x) ;
00044 */
00045

00049 **  The presence of the required functions are detected during the configure
00050 **  process and the values HAVE_LRINT and HAVE_LRINTF are set accordingly in
00051 **  the config.h file.
